# Contribuer à awesome-teif

Merci de vouloir contribuer. Ce dépôt vaut par la qualité et la fiabilité de ce qu'il référence — pas par son volume.

## Proposer une ressource

Ouvrez une **issue** ou une **pull request** avec :

- Le lien exact vers la ressource (documentation, outil, bibliothèque, article...)
- Une description courte (1-2 phrases) de ce qu'elle apporte et à qui elle s'adresse
- Si c'est un outil/bibliothèque : son état de maintenance (date du dernier commit, licence)

## Ce qui est accepté

- Documentation officielle (TTN, TunTrust, ANCE, normes ETSI/W3C/IETF)
- Bibliothèques et outils réellement utilisables pour valider, signer ou traiter du TEIF
- Retours d'expérience concrets (pièges rencontrés, cas métiers) — factuels, sans contenu confidentiel client
- Corrections de liens morts ou d'informations obsolètes (versions XSD, statut prod/bêta...)

## Ce qui n'est pas accepté

- Auto-promotion sans rapport direct avec l'écosystème TEIF
- Ressources dont l'exactitude ne peut pas être vérifiée
- Contenu dupliquant une entrée déjà présente sans valeur ajoutée

## Exactitude avant tout

Toute affirmation technique (statut d'une version XSD, comportement d'une norme, contenu d'un schéma) doit pouvoir être vérifiée à partir d'une source citée. En cas de doute sur un point, dites-le explicitement dans votre contribution plutôt que de l'affirmer.

## Code de conduite

Restez factuel et respectueux. Les échanges doivent servir les personnes qui utilisent ce dépôt pour résoudre un vrai problème technique.
